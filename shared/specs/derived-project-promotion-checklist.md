# 派生项目推进清单

当一个种子项目准备从“强化脚手架状态”推进到“较深实现状态”时，可参考以下清单：

## 继承基础底座

- 继承认证与权限基线
- 继承系统管理模块
- 继承仪表盘基础能力

## 增加业务模块

- 至少补充 2~4 个代表业务模块
- 更新菜单与路由
- 更新数据库 schema 与种子数据
- 更新项目 README

## 验证

- 后端 `mvn clean package`
- 后端 `mvn test`
- 前端 `npm.cmd install`
- 前端 `npm.cmd run build`
- 数据库初始化验证

## 文档同步

- 项目 README
- 模块说明
- 启动说明
- 扩展说明
- 交接文档中的项目状态
